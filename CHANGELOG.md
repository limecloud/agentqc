# Changelog

## v0.5.0 - 2026-05-10

- Added a dedicated testing-technique and composition guide covering snapshots, smoke tests, black-box, white-box, gray-box, replay, chaos, security/adversarial checks, semantic eval, and release install smoke.
- Added advanced evidence-braid recipes for runtime/UI, TUI approval, provider adapter, browser agent safety, channel gateway, scheduler recovery, skill/plugin lifecycle, and release confidence.
- Reframed current standard pages as a product-neutral protocol and kept product-specific material only in legacy compatibility, version-history, or explicit example pages.
- Updated navigation, README, source index, LLM entrypoints, public schemas, and examples for `0.5.0`.

## v0.4.0 - 2026-05-10

- Added Agent UI-inspired best practices, flow taxonomy, evidence contract, and performance/reliability metrics pages.
- Expanded quickstart, project classification, gate matrix, acceptance scenarios, and verdict guidance with runtime-backed surface evidence rules.
- Updated source traceability, navigation, README, and LLM entrypoints to include Agent UI, Agent Knowledge, Playwright, Vitest, pytest, and local star-project references.
- Bumped package and public plan schema version to `0.4.0`.

## v0.3.0 - 2026-05-10

- Expanded project case studies for representative runtime, TUI, gateway, scheduler, UI, release, and eval systems.
- Added interaction surface testing guidance for GUI, TUI, WebUI, desktop, browser automation, and mobile/channel flows.
- Added a Claude Code-style TUI/runtime example and expanded public QC plan samples for representative Agent project shapes and qcloop.
- Documented concrete evidence expectations per interaction surface instead of only listing coarse gate families.

## v0.2.0 - 2026-05-10

- Repositioned Agent QC from a product-specific draft to a portable Agent-project QC standard.
- Added project profiles for runtimes, SDKs, tool gateways, channel gateways, UI/TUI/desktop clients, skills/plugins, schedulers, distributions, and evals.
- Added a generic cross-project gate matrix and moved product-specific details out of the core scope.
- Added case-study patterns from local runtime, gateway, scheduler, UI, release, and eval repositories.
- Expanded schemas and examples for generic Agent QC plans and qcloop jobs.

## v0.1.0 - 2026-05-10

- Initial product-focused Agent QC draft standard.
- Added QC object model for plans, cases, gates, runs, verdicts, evidence, and reports.
- Added an initial product-specific gate matrix for local checks, contract tests, GUI smoke, Playwright, qcloop, and review.
- Added qcloop integration rules and public JSON examples.
- Added public schemas for `qc_plan`, `qc_case`, and `qc_verdict`.
- Added bilingual documentation and LLM entrypoints.
- Added GitHub Pages deployment workflow for the documentation site.
